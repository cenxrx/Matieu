Тесты для класса Service:
1. Название услуги не пустое — проверяет что поле Name заполнено и имеет длину больше 0.
2. Цена услуги положительная — проверяет что Price больше 0 и имеет тип decimal.
3. Категория услуги валидна — проверяет что Category равна "Custom" или "Cosplay".
4. Путь к изображению имеет правильный формат — проверяет что ImagePath начинается с "pr/" и содержит слэш.
5. ID коллекции может быть null — проверяет что CollectionId может быть пустым (услуга без коллекции).

Тесты для класса User:
1. Логин пользователя не пустой — проверяет что поле Login заполнено.
2. Роль пользователя валидна — проверяет что RoleName входит в список допустимых ролей (admin, moderator, user).
3. Баланс пользователя неотрицательный — проверяет что Balance >= 0.
4. Полное имя может быть установлено — проверяет что FullName заполнено и содержит пробел (имя и фамилия).
5. ID пользователя положительный — проверяет что Id больше 0.

Тесты для класса Collection:
1. Название коллекции не пустое — проверяет что поле Name заполнено.
2. ID коллекции положительный — проверяет что Id больше 0.
3. Коллекция может быть создана со значениями по умолчанию — проверяет инициализацию пустого объекта.
4. Название коллекции может содержать кириллицу — проверяет поддержку русских символов в названии.
5. Свойства коллекции могут быть изменены — проверяет возможность модификации полей объекта.
